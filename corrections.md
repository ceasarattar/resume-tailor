# corrections.md

This file is injected (in full) into every tailoring prompt. It starts with the
hard, non-negotiable rules below. Over time, corrections you submit in the UI get
appended under "Learned corrections" as structured lines.

## Hard rules (never violate)

1. NEVER invent experience, employers, job titles, dates, metrics, or anything not
   present in `profile/about-me.md` / `profile/experience.json`. Never claim
   anything listed under "Things I will NOT claim" in about-me.md.
2. Only surface skills I actually have. You may reorder and relabel skills so the
   JD's keywords appear first, but do not add skills I don't possess.
3. Weave the JD's priority keywords in naturally. NO keyword stuffing. NO hidden
   text, white text, or off-page text of any kind.
4. Rewrite the summary to mirror the target role using only true facts.
5. Reorder and rephrase experience bullets to foreground the most relevant impact,
   keeping real metrics. Do not fabricate or inflate numbers.
6. Output a COMPLETE, compile-ready, single-column LaTeX document. One page.
7. Standard section headings only (Experience, Education, Skills, Projects).
   No multi-column layout, no tables-for-layout, no icons, no text inside graphics.
8. When the JD requires something I appear to be missing, FLAG it in the missing-
   requirements list. Do not fake it to fill the gap.
